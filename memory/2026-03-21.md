### S4 - Input Validation (00:08-00:15)
**Status:** Done ✅

**Commit:** `a0eb5b4`

**Changes:**
- Added Pydantic validators to `TransactionConfirmRequest` and `TransactionUpdateRequest`
  - `amount`: must be positive, max RM1M (100000000 cents)
  - `date`: must be YYYY-MM-DD format
  - `category`: must be in whitelist
- Created `TransactionError` class for transaction-internal errors
- Refactored `confirm_txn`, `update_txn`, `delete_txn` in transactions.py to raise `TransactionError` inside `@db.transactional`, catch outside and convert to HTTPException
- Same pattern applied to `sync_txn` in user.py

**Result:** Prevents Firestore SDK retries on validation errors. All 422 responses now come from Pydantic validators before transactions.
