## Coding Standards Updates

### Rule #10: Parallel Agent File Ownership
- Every file has exactly one owner
- Agents read anything, write only owned files
- Interface contracts defined before parallel work
- Fixes verified by reading actual file content (don't trust agent reports)

### Rule #11: Phase Separation (Design → Plan → Develop → Test → Review → Fix)
- Minimum 6 phases per feature, each ≤20 min
- Each phase = separate agent spawn
- Hybrid session approach: Design→Plan and Review→Fix share sessions; Develop and Test get fresh sessions
- Context window dilution: focused agent > loaded agent

### Key Lessons from All Waves
- GLM-5 fix agents **lie about applying fixes** — always verify actual file content
- Mock-based tests give false confidence — real-service tests catch real bugs
- Define async/sync contract upfront — mismatch cost extra fix cycles
- 3 Kimi + 3 GLM-5 split: Kimi for boilerplate, GLM-5 for complex/review work
