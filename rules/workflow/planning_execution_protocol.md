# Planning Execution Protocol

Use this 10-step sequence for serious work.

1. **Review** — inspect existing files and patterns first
2. **Test-First** — define how success will be checked
3. **Standards** — look for existing conventions before inventing new ones
4. **Gap** — if no standard exists, choose the smallest good approach
5. **Implement** — do the work
6. **Test** — run the checks
7. **Audit** — look for stale docs, rough edges, follow-up cleanup
8. **Confirm** — make sure the result matches the request
9. **Cleanup** — only with user approval for anything destructive
10. **Update** — write state / diary / memory as needed

Do not skip straight from implementation to “done.”
