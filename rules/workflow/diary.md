# Diary Protocol

`plans/diary.md` is the append-only lab notebook for meaningful work.

## When to write

Write at the end of a task or session when:
- code/files changed
- a decision was made
- a lesson was learned
- a dead end matters

## Entry shape

```markdown
## YYYY-MM-DD — [task]

**Goal:** ...
**Outcome:** DONE | PARTIAL | BLOCKED | ABANDONED

### What happened
...

### Decisions
- ...

### Dead ends
- ...

### State at session end
...
```

Do not rewrite history. Prepend a new entry instead.
