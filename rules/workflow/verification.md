# Verification

Do not trust your own feeling that something is done.

## Loop

For each meaningful step:
1. implement
2. verify
3. if it fails, fix and retry
4. after 3 failed tries with the same approach, stop and change approach or ask

## Verification can be
- a test run
- a command output
- a file diff review
- a behavior check against the user request

## Rule

Never mark a step complete without a real check.
