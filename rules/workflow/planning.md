# Pre-Execution Planning

Never do non-trivial work without a plan.

## Mandatory Order

1. Read the relevant files
2. Define the real goal
3. Write a short, testable plan
4. Pause for approval when the change is risky or broad
5. Execute step by step

## A valid plan

A plan must be:
- specific
- testable
- ordered
- small enough to finish or split

## Session State

For work that may span multiple turns, maintain a `plans/*-session-state.md` file derived from `plans/SESSION_STATE.template.md`.

## Diary

For meaningful work, update `plans/diary.md` at the end with what happened, decisions, and next-step context.
