# Planning Patterns

## Lightweight Plan

Use for moderate work:

```markdown
# Plan: [task]
- [ ] Step 1
  - Verify: ...
- [ ] Step 2
  - Verify: ...
```

## Full Session-State Plan

Use when work may span turns or sessions:
- task
- status
- plan steps
- completed
- in progress
- decisions made
- failed approaches
- blocked
- next steps
- exact verify command/check

## Rule of Thumb

- one reply / one file / obvious fix → lightweight
- many files / uncertainty / likely interruption → full session state
