# State Tracking

## `state.md`

Use as the top-level view:
- In Progress
- Ready
- Backlog
- Done

## `plans/`

Use for live task state and handoff when the work needs detail.

## What to record

Record:
- task name
- next step
- blocker
- decisions that affect future work
- failed approaches worth not repeating

Do not record every tiny edit. Track meaning, not motion.
