# Workspace Anatomy

## Core Files

- `AGENTS.md` — workspace identity, startup rules, and operating expectations
- `state.md` — active/backlog/done task index
- `plans/SESSION_STATE.template.md` — reusable handoff template
- `plans/diary.md` — append-only work diary
- `memory/YYYY-MM-DD.md` — day-level continuity
- `MEMORY.md` — curated long-term memory

## Directories

- `skills/` — local operational guides
- `rules/workflow/` — execution discipline
- `tasks/` — optional artifacts or task-specific notes
- `memory/` — daily notes

## Practical Rule

Use the smallest number of files that preserves continuity. If a file does not help future-you resume work, question whether it should exist.
