# Evolution Workflow

## When to Improve the Harness

Improve it when:
- the same confusion repeats
- a missing file or rule causes drift
- resuming work is harder than it should be
- future-you would obviously benefit from one more constraint or note

## How to Improve It

1. Fix the immediate pain
2. Ask why the harness failed to prevent it
3. Add the lightest useful rule, note, or template
4. Record the reason in `plans/diary.md` or `memory/YYYY-MM-DD.md`

## Avoid

- adding structure because it feels sophisticated
- copying a whole framework when one paragraph would do
- keeping stale rules around after reality changed
